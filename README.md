# Phishout - Plateforme de CTF

Bienvenue sur **Phishout**, un challenge immersif autour de l’**OSINT** et de la **stéganographie**. Ce CTF est conçu pour tester vos compétences en recherche, analyse et résolution de problèmes numériques dans un contexte réaliste.

---

## 🎯 Objectifs

Les participants devront :

* Identifier et extraire des informations cachées dans différents médias (images, fichiers, réseaux sociaux…).
* Résoudre des énigmes de type **stéganographie** pour récupérer les flags.
* Démontrer leurs compétences en **OSINT** pour trouver des informations publiques mais difficiles à relier.

---

## 🧩 Description du challenge

Vous êtes plongés dans un univers fictif où un réseau de hackers vend des **starter packs de phishing**. À sa tête :

* **Gérard**, 34 ans, ancien développeur full stack devenu chef du réseau.
* **Marin**, 19 ans, passionné d’informatique.
* **Rémy**, 20 ans, également passionné d’informatique.

Votre mission : identifier leurs secrets et récupérer les **flags** cachés dans différents supports numériques.

---

## 🛠️ Contenu fourni

* **Ressource de départ** : [informations.rar](informations.rar)
* Fichiers image, texte ou autres médias utilisés pour cacher des flags.
* Une interface web pour tester vos flags.

---

## 📖 Règles

1. **Tous les flags suivent le format** `phishout{...}`.
2. Les flags ne doivent pas être partagés publiquement.
3. Les participants peuvent utiliser **OSINT**, stéganographie et analyse de fichiers.
4. Les flags doivent être soumis via l’interface web pour validation.

---

## 💻 Structure du site web

* **Accueil** : présentation de l’histoire et des personnages.
* **Liste des challenges** : cartes représentant chaque flag à découvrir.
* **Validation** : champ texte pour entrer les flags, avec retour immédiat si correct ou incorrect.
* **Notifications** : affichage de messages (succès, erreur, avertissement).

---

## 📚 Conseils pour les participants

* Inspectez tous les fichiers de manière méticuleuse.
* Analysez les métadonnées des images et fichiers.
* Faites attention aux formats cachés ou aux encodages non évidents (base64, hex, bits LSB…).
* Notez vos découvertes pour les soumettre correctement.
